# DAMA DMBOK2 - StudyVault

The DAMA-DMBOK2 is the industry-standard reference for managing data as an organisational asset — covering governance, architecture, quality, security, metadata, and more across 600 pages and 17 chapters.

These notes distil all 17 chapters into a **2–2.5 hour** read, compared to a **~13 hours** read of the original book. 

Most data professionals pick up this foundational knowledge piecemeal over years. These notes put it in one place, upfront.

**Who this is for:** Anyone working in or alongside data — governance, engineering, analytics, or BI. Useful at any experience level, but especially valuable earlier in your career when the foundations aren't yet second nature.

Not sure if the full book is worth your time? Start here.

These notes were generated by [Cortex Code](https://docs.snowflake.com/en/user-guide/cortex-code/cortex-code) (Snowflake's AI coding assistant) using the [tutor-setup skill](https://github.com/bevibing/tutor-skills). The skill processes source material and produces a structured Obsidian vault with study notes, practice questions, dashboards, and progress tracking, all designed for active recall and spaced repetition.

> **Warning:** These notes were AI-generated and may contain inaccuracies, misinterpretations, or missing nuance from the original text. If you spot something that doesn't look right, please open an issue or submit a PR with corrections. Your feedback helps improve the material for everyone.

## Getting Started

### 1. Get the notes

Clone or download this repository to your machine:

```bash
git clone https://github.com/tanyaeveritt/dama-dmbok2-book-notes
```

Or click **Code > Download ZIP** on GitHub and extract it. Put the folder wherever you like. Your Documents folder, Desktop, a dedicated study directory, etc. The location doesn't matter because Obsidian references it by path, not by a fixed location. You can move or rename the folder later and just re-open it in Obsidian.

### 2. Install Obsidian

[Obsidian](https://obsidian.md) is a free markdown-based note editor. You need it to view these notes with full formatting (collapsible answers, wikilinks, callouts, tags).

1. Download Obsidian from [obsidian.md/download](https://obsidian.md/download) (available for macOS, Windows, Linux, iOS, and Android)
2. Install and open Obsidian

### 3. Open the vault in Obsidian

1. In Obsidian, click **"Open folder as vault"**
2. Navigate to wherever you saved the `Data_Management_StudyVault` folder and select it
3. The vault opens and you can start navigating

If you ever move the folder to a new location, just use **"Open folder as vault"** again and point Obsidian to the new path. All internal links between notes are relative, so everything stays connected regardless of where the folder lives on your system.

**Note:** You can read the `.md` files in any text editor or on GitHub, but collapsible answer blocks (`> [!answer]-`), wikilinks (`[[Note Name]]`), and callouts won't render properly outside Obsidian.

## Interactive Quizzing with the Tutor Skill (Optional)

If you have access to [Snowflake Cortex Code](https://docs.snowflake.com/en/user-guide/cortex-code/cortex-code) or Claude with skill support, you can install the [tutor skills](https://github.com/bevibing/tutor-skills) to get an interactive quiz experience.

### Setup

1. Install the tutor skill into your Cortex Code or Claude environment following the instructions at [github.com/bevibing/tutor-skills](https://github.com/bevibing/tutor-skills)
2. Open a terminal session with Cortex Code (or Claude) in this vault's directory

### Running a quiz

Type `/tutor` to start an interactive tutoring session. Here's what happens:

1. **The skill scans the vault** - it reads the study notes, practice files, and your Learning Dashboard to understand what topics exist and how you've performed so far
2. **You choose a mode:**
   - **Diagnostic assessment** - a broad quiz across all chapters to identify your strengths and weaknesses
   - **Study a section** - focused review of a specific chapter or topic
   - **Drill weak areas** - targets topics where your Learning Dashboard shows low scores
   - **Check progress** - view your dashboard stats and get recommendations on what to study next
3. **The AI quizzes you interactively** - it pulls questions from the practice files, asks them one at a time, evaluates your answers, and gives feedback
4. **Progress is tracked** - after the session, the skill updates your `Learning Dashboard.md` and the concept tracker files in `concepts/` with your results. Topics are scored and colour-coded:
   - 🟥 Weak (0–39%)
   - 🟨 Fair (40–69%)
   - 🟩 Good (70–89%)
   - 🟦 Mastered (90–100%)
5. **Error notes are recorded** - when you get something wrong, the specific mistake and the correct answer are logged in the concept tracker so you can review them later

The tutor adapts to your performance over time. Weak areas get prioritised, and mastered topics are deprioritised, giving you an efficient study loop.
